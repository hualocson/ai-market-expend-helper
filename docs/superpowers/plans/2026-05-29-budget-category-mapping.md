# Budget ↔ Category Mapping Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give every budget a single owning category so a budget always resolves to exactly one category and a category can own many budgets, plus surface/edit that category in the budget form and card.

**Architecture:** Add a `category` text column to the `budgets` table (defaulting to `Other`, backfilling existing rows via the column default). Validate it with Zod at the route layer (`z.nativeEnum(Category)`), thread it through the typed `BudgetCreateInput` / `BudgetUpdateInput` payloads and the three budget read mappers, then wire it into the budget form (reusing `CategoryChipRow`) and show it as an icon-only badge on the budget card (reusing `ExpenseItemIcon`). `budget.category` is inert metadata — it never reads or writes `expense.category`.

**Tech Stack:** Next.js 15 App Router · Drizzle ORM (PostgreSQL) · drizzle-kit migrations · Zod · TanStack Query · React 19 · Tailwind v4 · Vitest.

**Reference spec:** `docs/superpowers/specs/2026-05-29-budget-category-mapping-design.md`

const asCreateInput = (value: Record<string, unknown>) =>

const asUpdateInput = (value: Record<string, unknown>) =>
